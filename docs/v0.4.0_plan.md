# v0.4.0 Plan

This document records the v0.4.0 CONCH milestone and its release scope.

## Current Status

- CLIP, PLIP, and optional CONCH are available through the model registry.
- CONCH uses the official `mahmoodlab/CONCH` package and the gated `MahmoodLab/CONCH` Hugging Face checkpoint.
- CONCH is optional. It is not installed as a core dependency and is not downloaded in CI.
- Report summaries and multi-run comparisons are available for saved retrieval, zero-shot, and prompt-sensitivity artifacts.
- Generated datasets, reports, demo patches, embedding caches, and model artifacts remain ignored through `.gitignore`.

## Milestone Scope

v0.4.0 adds optional CONCH support while keeping the project low-compute, patch-level, and CI-friendly.

Included:

- `CONCHWrapper` with the same public interface used by existing CLIP-style wrappers
- registry support for both `conch` and `MahmoodLab/CONCH`
- clear setup errors for missing optional dependencies, gated access, or unavailable weights
- local feasibility documentation for Hugging Face access, dependency installation, CPU loading, text/image embeddings, and CUDA smoke checks
- offline tests that use fake CONCH objects and do not download weights
- release notes and release-readiness documentation

Not included:

- CONCH as a required dependency
- bundled CONCH weights
- automatic Hugging Face credential management
- model downloads in CI
- committed generated comparison reports
- clinical diagnosis or clinical decision support claims
- full WSI high-throughput processing
- model training or fine-tuning
- dashboard or hosted experiment tracking

## External Constraints

As of May 6, 2026, public CONCH usage should be treated as gated and optional:

- The Hugging Face model page for `MahmoodLab/CONCH` describes the model as gated and lists a CC-BY-NC-ND-4.0 license.
- Access requires agreeing to usage terms with an approved Hugging Face account.
- The documented loading path uses the CONCH GitHub package and `create_model_from_pretrained`, not a plain `transformers.AutoModel` flow.

Primary references:

- Hugging Face model card: <https://huggingface.co/MahmoodLab/CONCH>
- CONCH GitHub repository: <https://github.com/mahmoodlab/CONCH>

## Implementation Notes

The wrapper keeps the existing model contract:

```python
encode_text(texts: list[str]) -> torch.Tensor
encode_images(images: list[Image.Image]) -> torch.Tensor
compute_similarity(image_embeddings: torch.Tensor, text_embeddings: torch.Tensor) -> torch.Tensor
```

The implementation:

- loads CONCH through `create_model_from_pretrained("conch_ViT-B-16", checkpoint_path="hf_hub:MahmoodLab/CONCH")`
- uses the package tokenizer object directly because the package helper `tokenize()` is incompatible with the installed tokenizer backend in the local test environment
- pads text token IDs to length `128`, matching CONCH expectations
- returns normalized CPU tensors from text and image encoding
- supports `device="cpu"`, `device="cuda"`, and automatic device selection
- leaves CLIP and PLIP behavior unchanged
