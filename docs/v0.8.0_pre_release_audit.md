## Scope Boundaries

v0.8.0 should remain patch-level and local-first. The release should not add or imply:

- whole-slide image file reading
- slide tiling or tissue detection
- slide pyramid rendering
- clinical diagnosis or clinical decision support
- tumor boundary identification
- required CONCH access
- Hugging Face token handling
- required model downloads in CI
- committed generated score CSVs, heatmaps, reports, datasets, model weights, or cache files

`score-coordinate-heatmap` loads a model and may download weights when they are not already cached locally. `render-coordinate-heatmap` remains the artifact-only path and does not load models or download weights.

## Data and Artifact Hygiene

The audit did not commit generated benchmark artifacts, real pathology images, model weights, Hugging Face caches, Python build metadata, or generated reports.

Ignored local artifact paths remain:

- `dataset/`
- `outputs/`
- `examples/demo_patches/`
- `.venv/`
- `.pytest_cache/`
- Python `__pycache__/` directories
- Python build metadata such as `*.egg-info/`, `build/`, and `dist/`

CONCH remains optional and gated for real inference. This audit did not configure Hugging Face tokens, install optional CONCH packages, or download model weights.
