# v0.6.0 MHIST Prompt-Grid Audit

Audit date: 2026-05-07

This note records a v0.6.0 reproducibility audit of the MHIST zero-shot prompt-grid workflow. It verifies that `run-zero-shot-grid` can run the committed prompt-grid config into a fresh output root, produce per-run reports, write logs, and generate a comparison summary without overwriting earlier local outputs.

This is an engineering and research observation, not a clinical result.

## Setup

Dataset:

- MHIST / Dartmouth Colorectal Polyps Histopathology Binary Classification Dataset
- Local dataset root: `dataset/MHIST/`
- Sampled test manifest: `dataset/MHIST/manifest_test_50_per_class.csv`
- Image root: `dataset/MHIST/images`
- Sample size used for this audit: 100 patches
- True label distribution: `HP=50`, `SSA=50`

Runtime:

- PathVLM-LiteBench version: `0.5.0`
- Command entry point: `pathvlm-litebench`
- Config: `configs/zero_shot_prompt_grid_mhist_sample.json`
- Runtime output override: `outputs/zero_shot_prompt_grid_mhist_sample_v060_audit`
- Device from config: `cuda`
- Frozen model inference only
- No fine-tuning
- No committed datasets or generated outputs

Models:

| Model key | Model | Notes |
|---|---|---|
| `clip` | `openai/clip-vit-base-patch32` | General CLIP baseline |
| `plip` | `vinid/plip` | Pathology-specific CLIP-compatible wrapper |
| `conch` | `MahmoodLab/CONCH` | Optional CONCH wrapper using gated Hugging Face access |

Prompt pairs:

| Prompt key | HP prompt | SSA prompt |
|---|---|---|
| `default` | `a histopathology image of hyperplastic polyp` | `a histopathology image of sessile serrated adenoma` |
| `diagnosis` | `hyperplastic polyp tissue` | `sessile serrated adenoma tissue` |
| `patch` | `a pathology patch showing hyperplastic polyp` | `a pathology patch showing sessile serrated adenoma` |

## Commands

Validate the committed prompt-grid config:

```bash
pathvlm-litebench validate-config \
  configs/zero_shot_prompt_grid_mhist_sample.json
```

Preview the run expansion without model inference:

```bash
pathvlm-litebench run-zero-shot-grid \
  --config configs/zero_shot_prompt_grid_mhist_sample.json \
  --output-root outputs/zero_shot_prompt_grid_mhist_sample_v060_audit \
  --dry-run
```

Run the grid:

```bash
pathvlm-litebench run-zero-shot-grid \
  --config configs/zero_shot_prompt_grid_mhist_sample.json \
  --output-root outputs/zero_shot_prompt_grid_mhist_sample_v060_audit
```

The run generated:

- 9 zero-shot report directories
- 9 `metrics.json` files
- 9 `predictions.csv` files
- 9 `errors.csv` files
- 9 `run.log` files
- `outputs/zero_shot_prompt_grid_mhist_sample_v060_audit/zero_shot_grid_comparison.md`

All generated artifacts are local ignored outputs and should not be committed.

## Reproducibility Findings

The prompt-grid workflow is reproducible in the current local environment:

- `validate-config` recognized 3 models, 3 prompt pairs, and 9 runs
- `--dry-run` expanded all report and log paths under the overridden output root
- the real run completed all 9 model/prompt combinations
- optional CONCH ran successfully in this environment
- the comparison summary was written under the overridden output root
- no generated dataset or output artifacts were staged for Git

The `--output-root` override solved the main workflow friction from the earlier baseline audit: a new grid run can now use the committed config without editing JSON or overwriting a previous local output directory.

## Interpretation

The results match the earlier sampled prompt-grid observation. Balanced accuracy remains close to chance across all 9 runs, and most runs show strong predicted-label bias.

Prompt wording changed the bias pattern:

- CLIP stayed HP-biased across all prompt pairs, though `patch` prompts reduced the HP share from 90-96% to 70%.
- PLIP changed direction: `default` and `patch` prompts were SSA-biased, while `diagnosis` prompts became HP-biased.
- CONCH was SSA-biased for `default` and `diagnosis`, while `patch` prompts produced a more mixed predicted distribution.

The useful engineering signal is that the prompt-grid command, output override, saved reports, logs, and comparison summary now work together as a reproducible local workflow. The useful research signal is that a single prompt pair is not enough to characterize model behavior on this MHIST zero-shot setup.
