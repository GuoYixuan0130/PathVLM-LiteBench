# v0.6.0 MHIST Reproducibility Audit

Audit date: 2026-05-07

This note records the first v0.6.0 reproducibility audit using the public patch benchmark workflow introduced in v0.5.0. It verifies that the documented MHIST sampled workflow can run locally for CLIP and PLIP, save reports, summarize reports, and compare saved report artifacts.

This is an engineering and research observation, not a clinical result.

## Setup

Dataset:

- MHIST / Dartmouth Colorectal Polyps Histopathology Binary Classification Dataset
- Local dataset root: `dataset/MHIST/`
- Sampled test manifest: `dataset/MHIST/manifest_test_50_per_class.csv`
- Image root: `dataset/MHIST/images`
- Image files present locally during audit: 3152
- Sample size used for this audit: 100 patches
- True label distribution: `HP=50`, `SSA=50`

Runtime:

- PathVLM-LiteBench version: `0.5.0`
- Command entry point: `pathvlm-litebench`
- Device selection: `auto`
- Resolved inference device: `cuda`
- Frozen model inference only
- No fine-tuning
- No committed datasets or generated outputs

Models:

| Model key | Model | Notes |
|---|---|---|
| `clip` | `openai/clip-vit-base-patch32` | General CLIP baseline |
| `plip` | `vinid/plip` | Pathology-specific CLIP-compatible wrapper |

CONCH was not rerun in this first v0.6.0 audit. It remains optional and requires gated Hugging Face access plus the optional CONCH package.

## Commands

CLIP baseline:

```bash
python examples/02_zero_shot_classification_demo.py \
  --manifest dataset/MHIST/manifest_test_50_per_class.csv \
  --image_root dataset/MHIST/images \
  --model clip \
  --device auto \
  --split test \
  --class_names HP SSA \
  --class_prompts \
    "a histopathology image of hyperplastic polyp" \
    "a histopathology image of sessile serrated adenoma" \
  --top_k 2 \
  --save_report \
  --report_dir outputs/zero_shot_clip_mhist_sample_v060_audit
```

PLIP baseline:

```bash
python examples/02_zero_shot_classification_demo.py \
  --manifest dataset/MHIST/manifest_test_50_per_class.csv \
  --image_root dataset/MHIST/images \
  --model plip \
  --device auto \
  --split test \
  --class_names HP SSA \
  --class_prompts \
    "a histopathology image of hyperplastic polyp" \
    "a histopathology image of sessile serrated adenoma" \
  --top_k 2 \
  --save_report \
  --report_dir outputs/zero_shot_plip_mhist_sample_v060_audit
```

Report summaries:

```bash
pathvlm-litebench summarize-report \
  --task zero-shot \
  --report_dir outputs/zero_shot_clip_mhist_sample_v060_audit

pathvlm-litebench summarize-report \
  --task zero-shot \
  --report_dir outputs/zero_shot_plip_mhist_sample_v060_audit
```

Report comparison:

```bash
pathvlm-litebench compare-reports \
  --task zero-shot \
  --report_dirs \
    outputs/zero_shot_clip_mhist_sample_v060_audit \
    outputs/zero_shot_plip_mhist_sample_v060_audit \
  --run_names clip_default plip_default \
  --output outputs/zero_shot_clip_plip_mhist_sample_v060_audit_comparison.md
```

Generated outputs are local artifacts under ignored `outputs/` paths and should not be committed.

## Reproducibility Findings

The v0.5.0 public patch workflow is runnable locally for the sampled MHIST CLIP and PLIP baselines:

- the sampled manifest loaded correctly
- `--device auto` selected CUDA
- reports were saved with `predictions.csv`, `errors.csv`, and `metrics.json`
- `summarize-report` generated per-run Markdown summaries
- `compare-reports` generated a two-run comparison Markdown file

The output naming was workable, but the command lines are long. A future quality-of-life improvement could add committed benchmark configs for CLIP-only and CLIP-plus-PLIP sampled MHIST baselines, so users do not have to copy long example commands.

## Interpretation

The useful result is not that PLIP is clinically better than CLIP. On this 100-patch sampled run, both models remain close to chance-level balanced accuracy and both show strong class bias in opposite directions.

The useful engineering signal is that the v0.5.0 workflow can reproduce saved local artifacts and comparison summaries for a real public patch dataset sample. The useful research signal is that default prompt wording is not enough to characterize model behavior on this task.

## Next Checks

Recommended follow-up work:

- add committed benchmark configs for CLIP and PLIP sampled MHIST zero-shot baselines
- run the existing prompt-grid config only after deciding whether to overwrite or parameterize its output root
- include CONCH in a v0.6.0 observation only when gated access and runtime are stable
- keep all datasets and generated outputs local and ignored
