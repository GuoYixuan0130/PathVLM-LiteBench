# v0.5.0 Post-Release Audit

Audit date: 2026-05-07

This document records a lightweight post-release check for PathVLM-LiteBench v0.5.0 after the tag and GitHub Release were created.

## Release State

- Latest GitHub Release: `v0.5.0`

- Local `HEAD` tag: `v0.5.0`

- Package version: `0.5.0`
- CI on release checklist commit: passed

## Local Smoke Checks

The following commands passed locally:

```bash
python -m pathvlm_litebench.cli version
python -m pathvlm_litebench.cli models
python -m pathvlm_litebench.cli demos
python -m pathvlm_litebench.cli validate-config configs/zero_shot_prompt_grid_mhist_sample.json
python -m pathvlm_litebench.cli run-zero-shot-grid --config configs/zero_shot_prompt_grid_mhist_sample.json --dry-run
python -m pytest tests
```

Observed results:

- `version` reported `PathVLM-LiteBench version 0.5.0`
- `models` listed CLIP, PLIP, and optional CONCH as implemented registry entries
- `validate-config` recognized the sample MHIST prompt-grid config as `zero_shot_grid`
- prompt-grid dry-run expanded 9 planned runs without model inference
- tests passed with 155 tests

## Editable Install Check

The local environment was not installed as a package before the audit, so the console script was initially unavailable. After running:

```bash
python -m pip install -e . --no-deps
```

the console script checks passed:

```bash
pathvlm-litebench version
pathvlm-litebench validate-config configs/zero_shot_prompt_grid_mhist_sample.json
```

This confirms the `pathvlm-litebench` entry point is available after an editable install. The audit also found that editable install metadata can create `*.egg-info/`, so `.gitignore` now excludes Python build metadata.

## Notes

- No model weights were downloaded during this audit.
- No real datasets or generated reports were committed.
- CONCH remains optional and still requires gated Hugging Face access plus the optional CONCH package for real inference.
