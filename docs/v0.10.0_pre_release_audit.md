# v0.10.0 Pre-Release Audit

Date: 2026-05-25

This audit records the local release-readiness checks for the v0.10.0 prompt-set patch-coordinate heatmap workflow release.

## Release Scope Reviewed

v0.10.0 includes:

- prompt-set config support for repeated prompt-scored coordinate heatmap runs
- `validate-config` support for `task: patch_coordinate_heatmap_prompt_set`
- `score-coordinate-heatmap-prompt-set --dry-run`
- model-free and image-loading-free prompt-set dry-run behavior
- model-backed prompt-set scoring over one coordinate manifest
- per-prompt `scores.csv`, `heatmap.png`, and `metadata.json` artifacts
- `prompt_key` metadata for prompt-set runs
- artifact-only prompt-set comparison CSV and Markdown summaries
- prompt-set workflow documentation
- v0.10.0 release checklist and release notes

v0.10.0 does not add whole-slide image file reading, slide tiling, tissue detection, slide pyramid rendering, clinical diagnostic claims, required CONCH access, Hugging Face token handling, bundled real pathology images, generated outputs, model weights, or local datasets.

## Documentation Audit

Reviewed documentation includes:

- `docs/v0.10.0_plan.md`
- `docs/prompt_set_coordinate_heatmap_workflow.md`
- `docs/release_checklist_v0.10.0.md`

- README release and milestone links

Documentation states:

- prompt-set dry-run does not load models or images
- prompt-set scoring may load models and may download weights if not cached
- CI uses fake models and does not download model weights
- comparison summaries read saved artifacts and do not load models or images
- generated datasets, outputs, score CSVs, heatmaps, metadata JSON files, comparison files, reports, caches, and weights should remain uncommitted
- heatmaps are patch-coordinate score visualizations, not clinical interpretations
- whole-slide image processing is outside the release scope
