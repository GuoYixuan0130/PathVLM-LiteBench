## v0.9.0 Scope Check

The v0.9.0 milestone goal was to improve reproducibility and comparison around prompt-scored patch-coordinate heatmaps while keeping the project patch-level, local-first, and lightweight.

Completed v0.9.0 work on `main`:

- Added run metadata output for `score-coordinate-heatmap`.
- Added `score-coordinate-heatmap --dry-run` validation without model loading.
- Added `metadata_output` support to `patch_coordinate_heatmap_scoring` configs.
- Added an artifact-only comparison command:
  - `compare-coordinate-heatmap-scores`
  - reads saved `scores.csv` artifacts
  - loads sibling `metadata.json` files when present
  - writes compact comparison CSV
  - optionally writes Markdown summary
  - does not load models, load images, or download weights
- Updated workflow docs, README CLI listings, and v0.9.0 plan status.

The core v0.9.0 scope is complete enough for release preparation.

## Boundary Check

The current v0.9.0 work remains within the intended project boundaries:

- `render-coordinate-heatmap` remains artifact-only and model-free.
- `score-coordinate-heatmap --dry-run` validates manifest and output paths without creating a model.
- `compare-coordinate-heatmap-scores` is artifact-only and does not load models or images.
- CI coverage uses fake model wrappers and saved artifacts, with no required model downloads.
- CONCH remains optional and gated.
- No Hugging Face token handling was added.
- No whole-slide image file reading was added.
- No slide tiling, tissue detection, or slide pyramid rendering was added.
- No clinical diagnostic interpretation was added.

Documentation wording remains focused on patch-level evaluation and patch-coordinate score visualization. Whole-slide image processing and clinical interpretation appear only as non-goals, limitations, or avoided wording.

## Artifact Hygiene
